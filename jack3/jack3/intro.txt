  Jack v3 — Complete System Description

  System Overview

  Jack v3 is an AI-powered intraday trading system for BankNifty (NSE index) built in jack3/. It combines 8 technical strategies with an LLM brain that
  generates a pre-market thesis and re-evaluates every 5 minutes. All AI decisions pass through hard-coded Python validation gates that cannot be bypassed.

  Market context: BankNifty futures, lot size 15, 5-min candles, 09:15–15:30 IST, force-close at 15:15.

  ---
  Architecture (7 Layers)

  config/settings.yaml     ← Central config (all tunable values here)
  data/                    ← Market data fetching & normalization
  indicators/              ← Plugin-based technical analysis (21 indicators)
  strategies/              ← 8 trading strategies with standard interface
  brain/                   ← LLM thesis + 5-min tick evaluator
  engine/                  ← Main loop, risk, state machine, broker, scorer
  journal/ + lab/          ← Journaling, backtesting, strategy ranking

  Strategies Layer (strategies/)

  Base classes (strategies/base.py):
  @dataclass
  class TradeSignal:
      strategy_name, direction, entry_price, stop_loss, target, confidence, reason, metadata

  @dataclass
  class ExitSignal:
      should_exit, exit_price, reason

  class Strategy(ABC):
      def check_entry(day_data, lookback, indicators, current_time, filters) → Optional[TradeSignal]
      def check_exit(position, day_data, current_time, current_price) → ExitSignal
      def score(signal, filters) → float  # 0.0–2.0

  Strategy constraints (hard rules):
  - Max 5 tunable parameters per strategy (anti-overfitting)

  AI / Brain Layer (brain/)

  ai_client.py — AIClient(provider, config)

  Three pluggable backends:
  1. anthropic: Direct Claude API (anthropic SDK)
  2. openai_proxy: OpenAI-compatible endpoint (Gemini via cliproxy at localhost:8080)

  thesis.py — ThesisGenerator

  Called at 08:55 AM. Returns:
  {
    "direction": "BULLISH"|"BEARISH"|"NEUTRAL",
    "confidence": float,           # 0.0–1.0
    "reasoning": str,
    "key_factors": [str],

    "expected_range_pts": int,

  Inputs: dependents dict + news research + last 5 days' journal + strategy rankings. Falls back to rule-based thesis if AI unavailable (max confidence:

  ---
  Engine Layer (engine/)

  loop.py — JackMainLoop (MAIN ORCHESTRATOR)

  Daily execution flow:
  08:45  run_pre_market()         → fetch dependents, research, generate thesis
  09:15  run_market_hours()       → tick loop every 5 min
           _run_one_tick()        → fetch candles → indicators → check SL/target → AI eval → validate → execute
  15:15  force_close_position()   → all positions closed
  15:35  run_post_market()        → journal, optional AI review

  Validation gates in _attempt_entry() (AI cannot bypass):
  1. Must be in entry-allowed trading phase
  2. No entries after 15:00
  3. SL capped to 1.5% max from entry
  4. Reward:Risk > 1.0 required
  5. risk.can_trade() check (daily loss, drawdown, trade count)

  Simulation mode: run_simulation_day(date, day_candles_df, pre_market_context) — reads from pre-loaded DataFrame instead of broker API. Used by historical
  simulation script.

  risk.py — RiskManager

  Position sizing formula:
  risk_amount = capital × max_risk_pct / 100
  quantity = floor(risk_amount / stop_distance / lot_size) × lot_size
  Capped at 30 lots. Min stop distance: 50 points.

  Full cost model per round-trip:
  entry_brokerage = entry_price × qty × 0.03% ÷ 100
  exit_brokerage  = exit_price  × qty × 0.03% ÷ 100
  stt             = exit_price  × qty × 1.25% ÷ 100  (sells only)
  gst             = (entry + exit brokerage) × 18%
  slippage        = 1 tick × 0.05 × qty
  total ≈ 0.4–0.5% of position per round-trip

  scorer.py — StrategyScorer

  Arbitrates when multiple strategy signals fire simultaneously. Score = strategy.score(signal, filters) × insight_weight × filter_mult. Confluence bonus
  (+0.15) if 2+ signals agree; conflict penalty (-0.1) if opposing.

  broker_dhan.py — DhanBroker

  - Paper mode: all orders simulated in memory
  - Live mode: Dhan REST API via v2 DhanContext initialization
  - Methods: place_order(), place_super_order() (entry+SL+target in one), modify_stop_loss(), get_ltp(), get_historical_intraday(), get_option_chain()
  - _mock_option_chain(): realistic paper mode chain built around current price level

  ---
  Lab Layer (lab/)

  backtester.py — backtest_strategy(strategy_name, candle_data, daily_data, config, params_override)

  Walks through historical 5-min candles day by day. Returns: win_rate, total_trades, avg_pnl, total_pnl, max_drawdown_pct, sharpe, trades_log,
  lookback_days.

  ranker.py — run_ranking(config, candle_data, daily_data)

  Ranking formula: score = win_rate × sqrt(total_trades) — balances accuracy with sample size.
  Saves to lab/rankings.json. Min 5 trades before a strategy is evaluated.

  discoverer.py — StrategyDiscoverer

  AI-proposes new strategies and validates them through the backtester before promoting.

  ---
  Learning Loop

  Day trades → Journal entries (JSON + Markdown)
           ↓
  Journal (last 10 days) → Fed into next day's thesis prompt
           ↓
  Strategy backtests → rankings.json (updated every 20 sim days)
           ↓
  Rankings → Fed into thesis prompt (top 3 strategies highlighted)
           ↓
  AI post-trade reviews → Lessons extracted → Fed into thesis
           ↓
  Better thesis → Better trades → Better journal (cycle continues)

  ---
  Key Design Decisions

  1. AI makes decisions, Python enforces them — AI cannot bypass SL/RR/phase/capital gates
  2. Full cost model — 0.4–0.5% round-trip prevents overstating backtest profits
  3. Paper mode by default — dhanhq package not required; install only when going live
  4. Plugin architecture — indicators and brokers are swappable
  5. Simulation reuses live code — run_simulation_day() uses the same _run_one_tick() as live trading
  6. Max 5 params per strategy — prevents overfitting
  7. Force-close at 15:15 — no overnight risk ever
  8. Minimum 5 trades before ranking — prevents luck from gaming rankings
